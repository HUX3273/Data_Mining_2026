3.2 数据流与预处理管道
预处理管道的核心原则是“先审计、后融合、再分析” ：对无法说明来源、口径或年份的数值不直接丢弃，而是保留为带
有不确定性标签的记录；只有经过冲突判定与可信度投票后的字段才进入最终人才梯队指标计算。
4 基线模型与核心算法实现
4.1 基线模型运行情况说明
• 所选基线方法：手动优先级融合 + 描述性统计。即对同一高校同一称号优先采用官方评估报告或高校人才办数据；
若缺失则参考第三方汇总；若多源冲突较小则保留最高优先级来源。
• 运行环境：Python 3.10/3.11，Pandas、NumPy、BeautifulSoup、Scikit-learn、Matplotlib/Seaborn。提交前
需在 README 中给出实际环境版本。
• 一键复现命令：
python src/preprocess.py --input data/raw --output data/interim/cleaned.csv
python src/audit.py --input data/interim/cleaned.csv --output results/audit_report.xlsx
python src/fusion.py --input data/interim/cleaned.csv --output data/processed/fused_talent.csv

4.2 核心进阶算法开发进度
本项目的进阶方案不是复杂模型堆叠，而是面向 Data-Centric 任务的数据审计与可信融合框架。 其核心创新点在于：把
多源人才统计数据拆解为“来源可信度、 统计口径、 时间戳、 冲突程度、 缺失模式”五类可解释因素，再用规则化可信度
投票得到可追溯的融合结果。

5 中期实验结果与阶段性分析
5.1 评估指标与测试集构建
• 评测数据集规模：以“双一流”高校为主体样本，围绕杰青、优青、长江学者、青年长江、万人计划等称号构建高
校—称号—年份级记录。提交前填写实际高校数量、称号类别数与原始记录数。
• 数据层指标：数据完整度、跨源冲突率、冲突解决率、口径不明比例、时间戳缺失比例。
• 模型层指标：K-Means 聚类轮廓系数、不同随机种子下聚类稳定性、核心指标对分型结果的敏感性。
• 业务层指标：能否形成可解释的高校人才梯队类型，例如“领军人才密集型” “青年储备型” “均衡发展型” “人才
规模薄弱型” 。
